# UX 三刀流：產品體驗設計知識庫

## 核心哲學

UX 設計不是追求極大化的使用者體驗，而是在**用戶需求、商業目標、數據驗證**三者之間找到動態平衡。這就是「三刀流」思維：

- **用戶流**：洞察人與產品互動的需求與感受，掌握心智模型
- **商業流**：站在使用者角度思考企業利益與資源配置，找到價值與成本的平衡
- **數據流**：用數據輔助設計決策，讓假設可驗證、讓迭代有方向

> 設計師所設計的不是人的「體驗」本身，而是人造物的「屬性組合」。人們與這些屬性互動時，產品間接影響了體驗。因此 UX 設計的本質是：**調控產品屬性，間接創造體驗**。

## 三刀流設計者的成長路徑

1. **入門（用戶流）**：專注 UX 方法與框架，以使用者至上的模式思考設計
2. **進階（用戶流＋商業流）**：將 UX 視為解決問題的邏輯，尋找用戶需求與商業目標的平衡
3. **資深（三刀流並行）**：懂得取捨完美設計換取資源最佳配置，運用數據輔助理性決策
4. **大師（彈性決策）**：經驗與方法並用，能主觀推論假設並用數據驗證，掌握產品全局

## 黃金法則（任何設計決策前先想）

1. **心智模型對齊**：使用者對產品的想像（心智模型）與產品實際的設計（概念模型）是否匹配？差距就是體驗問題的來源。
2. **不是設計師說了算**：設計師不是使用者。「我以一個使用者的角度和你說⋯」是最危險的話。用研究取代直覺。
3. **體驗 = 前 × 中 × 後**：使用前的期待、使用中的操作、使用後的回想，三段都是體驗。
4. **三維體驗**：每段體驗都包含功能面（有用嗎）、操作面（好用嗎）、情感面（喜歡嗎）。
5. **迭代是常態**：設計無法一次到位。假設 → 驗證 → 修正，每次靠近使用者一點。
6. **產品成功 = 公司成功**：產品到達越多使用者手上，就是設計的成功。商業可行性不是妥協，是必要條件。
7. **利益關係人都重要**：使用者只是其中之一。公司、品牌、合作方、通路、廣告主⋯都影響產品設計。
8. **數據不做決定，數據提供證據**：數據輔助決策，不取代設計思考。

## 快速參考：UX 設計流程全景

```
策略定向 → 問題探索 → 目標族群確認 → 需求分析 → 價值主張
    ↓
需求排序 → UX品質指標 → 產品藍圖/MVP → 商業模式
    ↓
資訊架構 → 人機互動設計 → UI設計 → 原型 → 測試
    ↓
設計評論 → PRD → 開發協作 → 上線
    ↓
進入市場策略 → 增長思維 → 全鏈路設計 → 首次體驗
    ↓
回訪動機 → 忠誠度 → 遊戲化 → 數據分析 → 使用性評估
    ↓
迭代優化 → Product-Market Fit → 產品生命週期管理
```

每個階段都需要三刀流並行思考：這對用戶好嗎？商業上合理嗎？有數據支持嗎？
