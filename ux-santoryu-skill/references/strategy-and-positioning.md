# 策略定向與市場定位

> 當你不確定產品方向、需要分析競爭環境、或定義目標族群時，使用這份指引。

---

## 1. 商業策略與 UX 策略的對齊

產品設計不能只從使用者出發。一個功能的決策至少涉及四個角度：

- **使用者角度**：這是用戶需要的嗎？
- **商業角度**：這要花多少錢？不做會損失多少客戶？
- **市場角度**：做了能提升市場競爭力嗎？
- **技術角度**：開發難度多高？排擠了多少其他功能？

**三刀流原則**：這四個角度不是互相矛盾，而是同一個產品的不同面。設計師需要找到平衡點，而非只站在使用者那一端。

### 策略制定步驟

1. 確認公司的商業目標（營收？用戶基數？市佔率？）
2. 將商業目標轉化為 UX 策略（什麼體驗能支撐這個目標？）
3. 評估可用資源與限制
4. 制定產品研發方向

### 公司目標影響設計

- **目標是營收**→ 定義收費功能、定價、付費/免費功能邊界
- **目標是用戶基數**→ 設計留存機制、社群功能、召回機制
- 不同目標下，同一個產品的設計規劃可能完全不同

## 2. 市場定位：客戶心理中的位置

市場定位不是你怎麼看自己的產品，而是**使用者腦中，你的產品佔據什麼位置**。

### 定位分析框架

1. **市場現況探索**：
   - 這群人在哪裡？
   - 他們真的有這個需求嗎？
   - 需求是否已被滿足？被誰滿足？
   - 我們的差異化在哪？

2. **競品分析的 UX 視角**：
   - 不只看功能對比，更要看**用戶在競品上的體驗痛點**
   - 客戶喜歡你的產品 ≠ 會買。客戶喜歡 ≠ 不喜歡競品
   - 競品分析要回答：用戶選擇我們而非對手的理由是什麼？

3. **定位策略**：
   - 找到競品未滿足的需求缺口
   - 用自家優勢填補該缺口
   - 在使用者心中建立明確的產品印象

## 3. 目標族群定義

### 常見錯誤

- 「所有年輕人」不是目標族群 — 就算是學生也有很大差異
- 目標太廣 = 沒有目標 → 行銷無法精準、設計無法聚焦

### 定義方法

**人口統計變數**（基礎）：年齡、性別、地區、收入、職業

**心理變數**（進階，更重要）：
- 生活態度、價值觀、興趣偏好
- 對產品類型的認知與使用習慣
- 痛點的強度與急迫性

### Persona 的正確使用

Persona 不是虛構角色的創作練習。它是：
- 基於真實研究數據的**需求代表**
- 幫助團隊對齊「我們為誰設計」的溝通工具
- 應包含：需求層次（學習、安全、愉悅、信任、獎勵等）

## 4. UX 投資報酬率（ROI）

當老闆問「多花資源在 UX 值得嗎？」

### 說服邏輯

1. **量化改善前的問題成本**：
   - 客服量、退貨率、流失率、轉換率低落的營收損失

2. **估計改善後的效益**：
   - 轉換率提升 X% 帶來的營收增長
   - 客服量下降節省的人力成本
   - 用戶滿意度提升帶來的口碑效益

3. **對比 UX 投入成本**：
   - 研究、設計、測試的時間與人力成本

### 關鍵觀念

- UX 投資是**風險管理**：在開發前發現問題，比上線後修復便宜 10-100 倍
- 用數據說話，不要只說「體驗很重要」— 這對商業決策者沒有說服力

## 5. 利益關係人分析

### 誰是利益關係人？

產品成功不只取決於使用者。從設計師到使用者之間存在：
- 公司 / 品牌（自有或代理）
- 合作方（內容、技術、流量）
- 通路平台（App Store、電商平台）
- 廣告宣傳管道
- 外送員、餐廳老闆等服務鏈參與者

### 分析原則

1. **識別**：列出所有與產品成功相關的角色
2. **定義利益**：每個角色期待從產品獲得什麼（錢、流量、品牌、數據、服務⋯）
3. **設計影響**：每個利益關係人的加入如何改變功能需求
   - 例：外送 App 加入「餐廳」→ 需要餐點上架、帳務、優惠系統
   - 再加入「外送員」→ 需要訂單分配、月結分紅、評價系統
4. **追求平衡**：提供合理利益，換取合理回報，建立穩固合作架構

### 利益不一定是錢

社群網站案例：使用者免費使用 → 產生行為數據 → 社群賣廣告給廣告主 → 廣告主接觸用戶 → 三方互為利益關係人。使用者的數據有時比錢更珍貴。

## 6. 產品價值主張

### 核心觀念

- **價值 = 產品為使用者解決的問題或滿足的需求**
- 錢是交換價值的媒介，不是價值本身
- 企業透過產品傳遞價值 → 客戶付費 → 企業獲得收入

### 價值主張設計

1. 明確定義：產品為誰解決什麼問題？
2. 差異化：為什麼用我們的而非競品？
3. 可感知：使用者能否在互動中真正感受到這個價值？

### 超越需求滿足

頂尖的設計不只滿足已知需求，更創造使用者「看到才知道自己需要」的新價值（如 Kindle、Clubhouse）。方法是巧妙組合人們已熟悉的心智模型元素，以低摩擦的方式傳遞創新。
