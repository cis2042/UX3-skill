# 產品迭代與生命週期

> 當你需要規劃產品迭代策略、處理使用者回饋、或思考產品長期發展時，使用這份指引。

---

## 1. 使用者回饋與迭代決策

### 回饋的來源

- **系統數據**：行為數據、漏斗轉換、錯誤日誌
- **客服回饋**：用戶主動反映的問題
- **使用性測試**：主動測試發現的問題
- **市場回饋**：競品動態、用戶評論、社群討論

### 不是所有回饋都要處理

**迭代取捨的三刀流思維**：

1. **用戶流**：這個問題影響多少用戶？嚴重度如何？
2. **商業流**：修復這個問題對營收/留存的影響？修復成本？
3. **數據流**：有數據支持這個問題的嚴重性嗎？

### Bug 也有優先級

不是所有 bug 都要馬上修：
- **嚴重 bug**（影響核心功能、造成數據損失）→ 立即修
- **中度 bug**（影響非核心功能、有替代方案）→ 排進下個 Sprint
- **輕度 bug**（外觀瑕疵、邊緣情境）→ 評估後決定
- 修 bug 也要考慮機會成本 — 修它等於不做其他事

### 迭代規劃原則

1. **數據先行**：用數據定義問題的優先級
2. **假設驅動**：每次迭代都有明確的假設要驗證
3. **最小改動**：每次改動範圍盡量小，便於歸因
4. **衡量效果**：上線後必須追蹤指標變化
5. **接受失敗**：有些迭代會失敗，重要的是學到了什麼

## 2. 運算式設計思考

### 傳統設計思考的盲區

- 高度依賴人的主觀判斷
- 便利貼投票不客觀
- 團體思維可能壓制少數但正確的意見
- 沒有量化基礎 → 難以追蹤進步

### 加入運算與數據

將設計思考的每個階段用數據強化：

| 設計思考階段 | 傳統做法 | 運算式做法 |
|---|---|---|
| 同理 | 訪談、觀察 | + 行為數據分析、用戶分群 |
| 定義 | 討論、彙整 | + 數據驗證問題的嚴重度和頻率 |
| 發想 | 腦力激盪 | + 競品數據分析、市場數據 |
| 原型 | 手作原型 | + 快速 A/B 測試 |
| 測試 | 使用性測試 | + 大規模行為數據追蹤 |

### 核心精神

不是用數據取代設計思考，而是用數據讓設計思考的每一步更可靠。直覺 + 數據 = 更好的決策。

## 3. 產品生命週期

### 五個階段

```
孕育 → 誕生/進入市場 → 成長 → 成熟 → 衰退 → 終結
```

### 各階段的設計師角色

**孕育期**（研發中）：
- 定策略、做研究、設計產品、驗證測試
- 這是前面所有章節在討論的主要階段

**進入市場**：
- 首次行銷策略
- 首次使用體驗設計
- 追求 Product-Market Fit

**成長期**：
- 增長設計
- 全鏈路思維
- 用戶留存與忠誠度

**成熟期**：
- 市佔率趨於穩定
- 競爭對手加入
- 需要持續創新避免衰退

**衰退/終結**：
- 用戶數據的安全處理
- 產品回收/退場策略
- 保持品牌形象的體面告別

### 不是每個產品都會經歷成功

多數產品在 PMF 之前就死了。新創失敗率高的原因就是無法證明市場需求。

## 4. S 曲線：對抗衰退

### 什麼是 S 曲線？

在產品成熟、尚未衰退之前，投入創新，創造第二波成長曲線。

### 兩種創新路線

**路線 A：優化現有產品**
- 改版、升級、新功能
- 讓產品更有競爭力面對市場
- 概念上等於「更好的同一個產品」

**路線 B：自我革命**
- 用核心優勢開創全新產品線
- 概念上是「取代自己的新產品」

### 案例

**Dyson**：
- 核心技術：旋風分離器（空氣動力學）
- 產品演進：吸塵器 → 直立式吸塵器 → 空氣清淨機 → 吹風機
- 同一核心技術，不同產品線，每條都是新的 S 曲線

**Netflix**：
- 1997：DVD 郵寄訂閱 → 500萬用戶
- 2007：線上影音串流 → 全球擴張
- 2013：自製影集 → 內容製作公司
- 每次在衰退前自我革命，用新技術/新模式延續品牌生命

### 給設計師的啟示

- 我們是最了解自己產品的人
- 不要等到衰退才行動 — 在成熟期就要開始規劃
- 創新不是推翻一切，而是在既有用戶基礎上找新路

## 5. 產品終結時的設計責任

### 終結不等於不管

- 用戶數據：如何安全提取或處置？要明確告知用戶
- 實體產品：回收策略（如 iPhone 以舊換新折扣）
- 品牌形象：體面的告別比突然消失好
- 替代方案：幫用戶過渡到新的解決方案

### 案例教訓

- 無名小站、MSN：曾經的高光時刻，被新產品（LINE、WhatsApp）取代
- Google Plus、Google Glass：未能達到 PMF 就終結
- 好的終結 = 保護用戶資產 + 維持品牌信譽

## 6. 為更美好的世界而設計

### Don Norman 的新使命（2023）

設計師應從「滿足用戶需求」提升到「解決人類全球性問題」。

三個核心主張：
1. **有意義的設計**：重視生活品質，而非只追求金錢回報
2. **可持續性**：考慮資源使用、能源消耗、生態影響
3. **以人類為本**：設計決策考慮全人類福祉，追求公平與包容

### UX → CX → HX 的格局提升

- **UX（使用者體驗）**：人與產品的互動感受
- **CX（顧客體驗）**：品牌、定價、購買流程的綜合感受
- **HX（人類體驗）**：個人與社會、環境、其他生命的整體互動感受

### 設計師的未來方向

- ESG 報告中的設計議題（流程、包裝、資訊揭露）
- 包容性設計 → 個人化設計（AI 技術使之可行）
- Web3 / 區塊鏈：追求更透明、可信任的數位體驗
- VR/AR / 元宇宙：全新的人際互動設計空間

> 設計師可以說是一種幫助世界進步的神聖職業。終其一生，在那條充滿不確定性的設計道路上繼續探索，為世界貢獻更美好的設計。
