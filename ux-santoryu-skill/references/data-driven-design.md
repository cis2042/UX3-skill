# 數據驅動的設計決策

> 當你需要用數據說服老闆、衡量產品成效、或將 AI 應用到產品設計時，使用這份指引。

---

## 1. 數據在 UX 設計中的角色

### 數據不做決定，數據提供證據

- 數據輔助設計師做更客觀的判斷
- 數據揭示「發生了什麼」，設計師判斷「為什麼」和「怎麼改」
- 完全依賴數據 = 失去設計洞察；完全不看數據 = 在猜

### 數據能做什麼

- 驗證設計假設是否成立
- 發現使用者行為模式中的異常
- 量化問題的嚴重程度（幫助排序優先級）
- 追蹤改版前後的效果差異
- 客觀比較不同方案（A/B 測試）

## 2. 關鍵數據指標

### 產品級指標

| 指標 | 意義 | 收集方式 |
|---|---|---|
| DAU / MAU | 活躍用戶數 | 後台數據 |
| 留存率（D1/D7/D30）| 用戶回來的比例 | 後台數據 |
| 轉換率 | 從瀏覽到行動的比例 | 埋點追蹤 |
| 跳離率 | 只看一頁就離開 | 網站分析 |
| 平均停留時間 | 用戶願意花多少時間 | 網站分析 |
| 平均瀏覽頁數 | 用戶探索深度 | 網站分析 |

### 功能級指標

- 功能使用率：多少人用了這個功能？
- 任務完成率：用了的人成功了嗎？
- 操作錯誤率：在哪裡犯錯？
- 操作時間：花了多久完成？

### 商業指標

- ARPU（每用戶平均收入）
- LTV（用戶終身價值）
- CAC（用戶取得成本）
- NPS（淨推薦值）

## 3. 數據收集設計

### 在產品設計時就規劃數據收集

不要等上線後才想「要追蹤什麼」。在 PRD 階段就定義：

1. **這個功能的成功指標是什麼？**
2. **需要收集哪些數據來衡量？**
3. **數據埋點要放在哪裡？**
4. **數據要怎麼呈現給團隊？**

### 漏斗分析

在關鍵流程的每個步驟埋點，追蹤轉換率：

```
訪問著陸頁 → 註冊頁 → 完成註冊 → 首次操作 → 核心功能 → 回訪
  100%         60%       35%        25%        18%       10%
```

每個環節的流失都是設計優化的機會。**流失最嚴重的環節 = 最該優先處理的。**

### 數據後台的價值

- 每天盯著績效後台 → 快速發現異常
- 觀察使用者在哪裡下錯指令 → 發現介面問題
- 按時段分析使用行為 → 理解用戶使用情境
- 比較每日增減趨勢 → 判斷改版效果

## 4. 使用性工程

### 什麼是使用性（Usability）？

使用者能否「有效、有效率、且滿意地」完成任務。

### 使用性評估方法

**專家評估**：
- 啟發式評估：專家依據設計原則逐一檢查
- 認知走查：模擬使用者執行任務的認知過程

**使用者測試**：
- 觀察真實使用者操作
- 記錄：任務完成率、錯誤率、完成時間、主觀滿意度
- 5 位使用者 ≈ 發現 85% 問題
- 迭代測試比一次大測試更有效

### 客服回饋不夠具體時怎麼辦？

- 「客戶說不好用」→ 多少客戶？哪些功能？什麼操作？
- 客服應該收集：具體功能、操作步驟、螢幕截圖
- 不接受模糊的「不好用」→ 要求具體化才能改善

## 5. AI 與 UX 設計

### AI 輔助 UX 設計

- **用戶研究**：自動分析大量回饋文字、情感分析
- **行為分析**：自動識別使用行為模式和異常
- **設計生成**：AI 輔助產生設計方案（但需設計師判斷）
- **A/B 測試優化**：多臂賭博機演算法自動分配流量

### AI 加入產品

- **個人化推薦**：根據用戶行為推薦內容/產品
- **智慧客服**：自動判斷問題類型、自動回應
- **預測性設計**：預測用戶下一步需求，提前準備

### UX 設計師需要了解的 AI 概念

- 不需要會寫程式，但要理解：
  - AI 能做什麼 / 不能做什麼
  - 數據品質如何影響 AI 表現
  - AI 的判斷可能有偏差（bias）
  - 使用者對 AI 的信任度需要被設計

## 6. Product-Market Fit 成效報告

### 何時需要這份報告？

產品上線一段時間後，向管理層/利害關係人彙報：產品在市場上的表現如何？

### 報告內容結構

1. **市場概況**：目標市場規模、成長趨勢、競爭格局
2. **用戶數據**：用戶數、活躍度、留存率、成長趨勢
3. **產品使用數據**：功能使用率、任務完成率、關鍵流程轉換
4. **商業表現**：營收、付費轉換、ARPU
5. **用戶回饋**：NPS、滿意度、常見痛點
6. **競品對比**：我們 vs 競品的關鍵差異
7. **下一步行動**：優化方向、迭代計畫

### 報告原則

- 用數據說話，不用感覺
- 好消息和壞消息都要呈現
- 每個數據後面要有「so what」— 這代表什麼？我們要怎麼做？
