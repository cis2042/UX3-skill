# 產品增長與用戶留存

> 當產品上線後需要獲取用戶、提升留存率、設計忠誠度機制、或規劃遊戲化體驗時，使用這份指引。

---

## 1. 進入市場策略（Go-To-Market）

### 關鍵問題

產品上線不等於成功。上線後要回答：
- 首批用戶從哪裡來？
- 他們為什麼要試用？
- 試用後為什麼留下來？

### 策略要素

1. **首批用戶來源**：廣告、內容合作、社群推廣、口碑推薦
2. **降落體驗設計**：用戶從廣告/連結點進來那一刻的體驗
3. **首次使用引導**：註冊 → 第一個成功操作 → 「啊哈時刻」
4. **價值感知速度**：用戶多快能感受到產品的核心價值？越快越好

### Product-Market Fit

- 產品真正被市場接受，才算存活
- 指標：留存率穩定、用戶自然增長、口碑推薦出現
- 多數新創死在 PMF 之前 — 這是產品最大的生死關

## 2. 產品增長思維

### 增長不只是行銷的事

傳統思維：產品做完 → 交給行銷去推廣
三刀流思維：**產品設計本身就是增長的引擎**

### AARRR 海盜指標

| 階段 | 指標 | 設計關注點 |
|---|---|---|
| Acquisition（獲取）| 新用戶數 | 廣告、降落頁、SEO |
| Activation（啟動）| 首次使用完成率 | 註冊流程、引導、啊哈時刻 |
| Retention（留存）| 回訪率 | 回訪動機、推播、價值持續供給 |
| Revenue（營收）| 付費轉換率 | 付費點設計、定價策略 |
| Referral（推薦）| 推薦率 | 分享機制、推薦獎勵 |

### 增長設計原則

- 每個環節都是一個可優化的體驗設計機會
- 用數據找到**流失最嚴重的環節**，集中火力改善
- 小幅迭代 > 大幅改版（風險更低、驗證更快）

## 3. 全鏈路思維

### 什麼是全鏈路？

使用者不是只在你的 App 裡活動。他們跨越多個接觸點：
- 社群媒體看到推薦
- Google 搜尋到你的產品
- 點擊廣告進入降落頁
- 下載 App → 註冊 → 使用
- 收到推播通知回來使用
- 在社群分享給朋友

### 設計原則

1. **一致的品牌體驗**：不管從哪個接觸點進來，感受要一致
2. **流暢的轉換**：從 A 接觸點到 B 接觸點的過渡不能斷裂
3. **情境感知**：
   - 從廣告來的 → 強調廣告承諾的價值
   - 從朋友推薦來的 → 強調社群信任
   - 從搜尋來的 → 提供他搜尋的解答

4. **裝置差異**：手機、平板、桌機的介面和互動方式不同

## 4. 首次使用體驗（First-Time UX）

### 為什麼重要？

首次體驗決定使用者是否留下。第一印象失敗 → 很可能永遠流失。

### 設計要點

1. **降落頁**：
   - 3 秒內讓用戶知道「這是什麼」「對我有什麼好處」
   - 清楚的行動呼籲（CTA）
   - 不同來源的用戶可能需要不同的降落頁

2. **註冊流程**：
   - 能簡就簡 — 第一次不要問太多資料
   - 可以在購買時再收集完整個資（用戶有動機時才願意填）
   - 利用瀏覽器自動填寫功能減少摩擦

3. **引導設計（Onboarding）**：
   - 帶使用者完成第一個核心操作
   - 讓他們感受到核心價值（啊哈時刻）
   - 漸進式引導，不要一次丟太多資訊

4. **首次體驗的成功指標**：
   - 註冊完成率
   - 第一個核心操作完成率
   - 首日/首週留存率

## 5. 回訪動機與忠誠度

### 新客 vs 老客的價值

- 開發新客戶的成本通常是留住老客戶的 5-25 倍
- 老客戶的終身價值（LTV）遠高於新客戶
- 因此：**留存比獲取更重要**

### 回訪動機設計

1. **內容更新**：持續提供新的有價值內容
2. **社群連結**：讓用戶與其他用戶建立關係
3. **進度感**：讓用戶看到自己的成長或積累
4. **推播提醒**：適時、有價值的通知（不是騷擾）
5. **習慣養成**：融入用戶日常行為模式

### 忠誠度迴圈

設計一個**讓用戶持續回來**的循環機制：

使用產品 → 獲得價值 → 累積資產（數據/等級/社群關係）→ 離開成本增加 → 繼續使用

### RFM 模型

分析用戶與產品的關係程度：
- **Recency（最近一次）**：多久沒回來？
- **Frequency（頻率）**：多常使用？
- **Monetary（金額）**：消費或互動的深度？

依 RFM 分類用戶，對不同類型採取不同策略：
- 高 R 高 F 高 M → VIP，維護關係
- 高 R 低 F 低 M → 新用戶，引導深入使用
- 低 R 高 F 高 M → 流失風險，啟動召回
- 低 R 低 F 低 M → 已流失，評估是否值得召回

## 6. 遊戲化設計

### 什麼是遊戲化？

把遊戲令人覺得好玩的元素，應用到非遊戲的產品中，增強使用者的動機和參與度。

### 好玩的本質

人覺得好玩，來自於：
- **挑戰與能力的匹配**（心流狀態）：太簡單無聊，太難挫折
- **自主感**：使用者覺得自己有選擇權
- **成就感**：完成目標的滿足
- **社交性**：與他人互動、競爭、合作
- **驚喜**：不可預期的獎勵

### 遊戲化設計框架

1. **目標系統**：給使用者明確的目標（每日任務、成就徽章）
2. **進度系統**：讓使用者看到自己的進展（等級、進度條）
3. **獎勵系統**：完成目標後的回報（積分、解鎖、特權）
4. **社交系統**：排行榜、團隊任務、分享
5. **反饋系統**：即時的正向回饋（動畫、音效、祝賀）

### 遊戲化的陷阱

- 不要為了遊戲化而遊戲化 — 如果核心產品沒有價值，加遊戲化也沒用
- 外在獎勵可能削弱內在動機 — 不要過度依賴積分和獎品
- 排行榜可能傷害落後的用戶 — 考慮分層或隱藏排名
